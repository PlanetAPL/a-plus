The Button Display Class

This is the display class for functions. The display is a labelled button, and when the button is pressed the associated function is executed. (To press a button, move the mouse cursor onto it and click the left button on the mouse.) The function must be monadic or niladic. Specifically, let fn be a monadic function and define aû(fn;data). Then the object a can be bound to the class button, and whenever the button is pressed, fn{data} is executed. A second form for defining a button is aû<{fn}, which is equivalent to aû(fn;) for monadic functions; when the button is pressed, the expression fn{()} or, for a niladic function, fn{} is executed. A dependency of the form a:<{fn} can also be bound to the button class; the value of this indirection is that the button will always invoke the latest definition of fn, since a change to fn will invalidate a and cause <{fn} to be evaluated again.

Note that the value of a variable bound to the button class is never a function, but rather is or includes a function array.
